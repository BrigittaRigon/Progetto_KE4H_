Data Acquisition and Validation Historical and architectural metadata were collected, including: construction in the 12th century, Bolognese dialect name “Porta ed San Mamel”, addition of a drawbridge in 1334, demolition in 1903, and building material brick. SPARQL SELECT queries confirmed that Porta San Mamolo was absent from the ArCo knowledge graph (KG), validating the need for structured integration. RDF Modeling via SPARQL CONSTRUCT CONSTRUCT queries were generated to encode key properties: Construction period → arco-cons:hasConstructionDevelopment Material → arco-den:hasMaterial Related event (drawbridge) → arco-event:hasRelatedEvent Descriptive information (dialect name) → arco-core:describes Demolition date → arco-core:hasNote Queries were expressed with canonical ArCo URIs and appropriate language tags (@en, @bo), producing RDF triples compatible with Virtuoso and other triple stores. SPARQL Verification Queries ASK queries were used to validate the presence of demolition information, returning false. DESCRIBE queries confirmed the absence of Porta San Mamolo in the KG, demonstrating the need for insertion. LLM-Assisted Query Generation ChatGPT produced CONSTRUCT queries via one-shot prompting. Gemini generated structured RDF triples using zero-shot prompting, using precise ArCo ontology terms. Outcome Porta San Mamolo’s historical, architectural, and descriptive data have been formally represented in RDF, ready for ingestion into the ArCo KG.
